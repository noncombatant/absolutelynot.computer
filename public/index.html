Hardware Vulnerability Bibliography

This selected bibliography focuses on exploitable vulnerabilities in computing hardware that arise from architectural, microarchitectural, or physical problems. (So, not all errata.) These vulnerabilities render even hypothetically-perfect software attackable: for example, even if the software has no spatial memory unsafety bugs, microarchitectural side-channels can provide an arbitrary read primitive, and Rowhammer can provide an arbitrary write primitive — just as if there were spatial memory unsafety. These attacks typically work across hardware-defined privilege boundaries.

This bibliography is inspired by the anonbib, a bibliography of work on anonymity systems. You might also enjoy that, since it’s great.

SMASH

“SMASH is a new JavaScript-based attack that gives the attacker an arbitrary read and write primitive in the browser.” Bypasses the Target Row Refresh (TRR) mitigation, even from JavaScript (i.e. without the direct ability to flush the cache, such as with the CLFLUSH instruction).

  • PoC on Github
Half-Double

Shows that as process scale goes down, the ‘range’ of hammering increases, making Rowhammer attacks a bit more practical: the ‘aggressor’ row need not be immediately adjacent to the ‘victim’ row(s).

  • Paper
  • More generally, the team’s HammerKit: tools for exploiting Rowhammer on Chrome OS systems
TRResspass

This work bypasses the Target Row Refresh (TRR) mitigations (various vendors have different forms of it) incorporated into DDR4 RAM. This work is a precursor to SMASH.

SmashEx

“SmashEx is an attack that exploits re-entrancy vulnerabilities in the exception handling designs of Intel SGX (Software Guard eXtensions) enclave runtimes. It is powerful enough to allow an attacker to read secrets or perform arbitrary code execution inside the victim enclave.”
